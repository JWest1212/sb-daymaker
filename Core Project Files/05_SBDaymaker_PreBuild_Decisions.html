v9 — the three-section cut (current; governs everything below)

SB Daymaker is now three sections: Explore · Saved · Discover SB. The Map screen and the My Plan itinerary builder are removed. In their place: Near Me is an in-view sort (geolocation → neighborhood anchor) on Explore and Saved; sharing is a view-only saved-list link (Option A — shared_states.kind='shared_list'; one item or a multi-select batch; the opener saves their own copy; no recipient PII); One Perfect SB Day is an Explore card that seeds the saved list; Discover SB holds neighborhood + theme guides that surface live happenings. Mapbox leaves the V1 stack (full map → Phase 2); the data contract is sbdaymaker_schema.sql. Where v9 conflicts with any decision recorded below, v9 governs. Status: v9 canon · 2026-06-21.

Earlier layer — revised by the UX decisions

These 26 decisions remain the build-ready resolution of the audit. A subsequent UX-decision layer (16 Locked Decisions + 7 follow-ons; rationale in Document 6) revised several of them, and v9 (above) revised several again. Where they conflict, v9 first, then the UX decisions govern; this record is preserved as written. Key revisions:

• B5 (My Plan) → retired in v9. The live app stays fully deterministic, but the My Plan builder, the optional timeline, the .ics export, and the drum-roll picker are all removed. The surviving social artifact is the view-only saved-list share-link (Option A).

• B8 (on-device save durability) → magic-link save-restore added. A passwordless email link (the shared_states token store) restores saves on any device, beyond the PWA-install hedge.

• A6 (email) → promoted to first-class, 2×/week (midweek + weekend), covering what's new and what's coming up. Web push for installed users is a committed Phase-2 fast-follow.

• Spine, tags, scope. Place-first → a three-tier happenings cascade (happening_tier + recurring_schedules); six personas → ten occasion tags (thing_tags); Near Me is an in-view sort (no Map screen) and Happy Hour is a "last confirmed" list (full map + live countdowns → Phase 2).

• Data contract. The schema is sbdaymaker_schema.sql, carrying these changes (incl. the Discover SB guide kind/zone and the shared_list share). The cost model (B12), sourcing posture (B3), pipeline (B2), and accessibility floor are carried forward unchanged.

04 — Resolving B12

The re-derived cost model

The old "$0–40/mo, mostly free" headline assumed free tiers plus the Google $200 umbrella credit — which ended Feb 28, 2025. Here's the honest picture. The confirmed Place Photo SKU ($7/1k after 1k free) means the variable photo curve still holds; the change is the fixed platform floor for a reliable build.

TierFixed platformVariable (photos + AI)Notes
Early devVercel Pro ~$20~$0Supabase still free; minimal traffic
Launch (pre-traffic)~$45 (Vercel + Supabase Pro)~$1–10Supabase Pro on → backups + no pausing
~100 weekly~$45~$43 photos + ~$5 AI≈ $93/mo all-in
~500 weekly~$45~$245 photos + ~$10≈ $300/mo all-in
~1,000 weekly~$45~$497 photos + ~$15≈ $557/mo — before the operator-photo program bends photos down

Still remarkably cheap, and still one-person-operable — just not literally "$0." The thesis survives; the framing is now accurate. Geocoding, weather, and email all sit inside free tiers at these scales; browser geolocation (the V1 Near Me anchor) is free, and Mapbox is a Phase-2 line item, not a launch cost.

05 — Resolving C5

Design system, consolidated

The tokens now live in one file — sbdaymaker_tokens.css — instead of being implicit in the wireframe CSS. It carries the palette with AA-safe text variants, the type and 4px spacing scales, radius/shadow, the motion spec (with a reduced-motion override), breakpoints, and a component-variant contract for buttons, cards, sheets, the save-heart, and badges. Build against the semantic tokens (--text, --surface, --accent) so the contrast rules from C2 stay enforced automatically.
