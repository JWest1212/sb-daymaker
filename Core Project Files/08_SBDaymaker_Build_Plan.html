How every build step works

Read this once — it's the rhythm of the whole build. From Phase 1 on, every phase is the same five moves, so you never have to wonder "what do I actually do?"

  1. Open your project in VS Code. Open the VS Code app. The first time, point it at your project: File menu → Open Folder → choose your sb-daymaker folder. After that it remembers.
  2. Open Claude Code. In VS Code's left-hand strip of icons, click the Claude icon. A chat box appears — that's your builder.
  3. Paste the phase's kickoff prompt and press Enter. Claude shows a plan (a numbered list of what it'll do). Read it. If it mentions a Map or My Plan feature (both removed), say "that's removed in v9, re-read CLAUDE.md and revise." Otherwise approve it (type "yes, go ahead" or click Approve).
  4. Let it build; answer its questions. It may pause to ask you to paste a key or confirm a choice. Do it, and it continues.
  5. Run the app and look at it. Claude starts a "development server" and shows a web address like http://localhost:3000. Ctrl-click (Cmd-click on Mac) it, or paste it into your browser — the app opens. That's how you "look at" the app in every checklist. To stop it, click the terminal area and press Ctrl + C.

Two safety nets, always there: Plan mode (make Claude show the plan before touching anything) and rewind (if a change breaks something, say "undo the last change" and you're back to when it worked). You can't permanently break anything.

Phase 0 · all YOU · click-by-click

Set up your tools and accounts

Every account and program exists, you're signed in, and a blank "coming soon" page is live on the internet. This is the most clicking — it gets much smoother after this.

0.1 — Create a GitHub account (where your code is stored)
  1. In your browser, go to github.com. Click Sign up; enter an email, password, username; verify your email.
  2. Click the + icon (top-right) → New repository.
  3. Name it sb-daymaker, leave the rest as-is, click the green Create repository. Leave the tab open.

✅ Check: you see a page titled sb-daymaker.

0.2 — Install Node.js (the engine that runs your app on your computer)
  1. Go to nodejs.org. Click the big LTS download button (the stable version).
  2. Open the downloaded file; click through Continue / Next → Agree → Install → Finish, accepting defaults.

✅ Check: Claude Code confirms this for you in step 0.7 — nothing to verify by hand.

0.3 — Install VS Code (where you'll see your project and talk to Claude Code)
  1. Go to code.visualstudio.com; download for your computer; install (drag to Applications on Mac; Next/Next/Finish on Windows).
  2. Open VS Code. A welcome screen appears — leave it open.

✅ Check: the VS Code app opens and shows a "Welcome" tab.

0.4 — Install Claude Code & sign in (your AI builder)

Requires a paid Claude plan. If you don't have one, go to claude.com and upgrade to Pro first.

  1. In VS Code's far-left icon strip, click the Extensions icon (four squares). A search box appears.
  2. Type Claude Code. Find the result published by Anthropic and click Install.
  3. A Claude icon appears in the left strip. Click it → click Sign in. Your browser opens; log in to your Claude account and approve. Return to VS Code.
✅ Check it worked
  • A Claude chat panel is visible inside VS Code.
  • It shows you as signed in (no more "Sign in" button).
  • If anything's off, type in the panel: "Are you set up correctly? Run a diagnostic."

Alternative if the extension gives trouble: a graphical Claude Desktop app at claude.com/download, or a one-line terminal installer on the official setup page. The extension is simplest for most people.

0.5 — Create your Supabase account & project (your database)
  1. Go to supabase.com → Start your project. Easiest: Continue with GitHub.
  2. Click New project. Name it sb-daymaker; create a database password and write it down somewhere safe; pick the region closest to California; click Create new project.
  3. Wait 1–2 minutes for setup. Leave the tab open.

✅ Check: you land on a project dashboard with a left menu (Table Editor, SQL Editor…).

0.6 — Create your Vercel account (where your app lives online)
  1. Go to vercel.com → Sign Up → Continue with GitHub and approve.

✅ Check: you see the Vercel dashboard.

Phase 8 · the engine room

The nightly autopilot & your approval screen

Automation that keeps the app fresh with almost no daily work — it gathers content, has Claude write blurbs/tags overnight, and shows it to you to approve in one tap. This is Document 9, Stage 2, automated.

YOU — set up the app's own AI key and the schedule:

  1. The app's Claude key (different from your Claude Code login): go to platform.claude.com (the Anthropic Console), sign in, click API Keys → Create Key, copy it (starts sk-ant-). Paste to Claude Code as a secret when asked. Billed per use, separate from your subscription.
  2. Your Supabase secret key: Supabase → Project Settings → API Keys → copy the Secret key (sb_secret_…; or the legacy service_role key if that's what you see). This one is powerful — only paste it where Claude tells you (server-side), never into a public place.
  3. Turn on the schedule: Claude will have you set your Vercel project to the Pro plan (needed for scheduled jobs) and will set the nightly timer.
🤖 Kickoff promptRead CLAUDE.md, the architecture doc, Document 9, and Phase 8. Build the nightly pipeline: one scheduled job that gathers facts, batch-enriches them with the Claude API on the cheapest model to add blurbs/tags/tier/category — using ONLY the facts provided, never inventing places, dates, or times — and stages them for review. Then build an admin "cockpit" screen where I approve or reject each item in one tap before it goes live. Plan mode first.

When something goes wrong (it will — that's normal)

  • A red error appeared: select the whole message, copy it, paste it to Claude Code, type "this errored — please fix it." Repeat until clean. You don't need to understand it.
  • A change made things worse: type "undo your last change" or use Claude Code's rewind to go back, then retry with a clearer request.
  • Claude built the wrong thing (a Map or My Plan): "that's removed in v9 — re-read CLAUDE.md and redo the plan."
  • A website's buttons don't match this guide: describe what you see to Claude Code and ask it to walk you to the right spot.
  • Unsure a phase is done? If any box is unticked, it's not. Don't move on.
