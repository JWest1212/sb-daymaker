Explore mobile readability — component reference

The build spec (SBDaymaker_Explore_Phase6_Build.md) references these by anchor. Sections A–C are the individual components with their pixel specs; D–E show them assembled at phone width. Thumbnails are gradient stand-ins; the spine node is a plain terracotta dot — both use live assets/tokens in the build.

Changed in rev 2: the row-card actions moved out of the top-right corner into a bottom meta row beside the date. This removes the empty gap under the title and lets the title use the full card width — no more mid-word truncation, and no need for a “stack actions on mobile” rule.

Reference A · shared component

Row card — full-width description (Option B)

The single row card used by This Week rows, Today briefs, and Tier-2/3. Thumbnail + occasion pill + title form a compact header row; the description spans the full card width at 3 lines; the date and the heart/share sit together in a bottom meta row. Change the ListCard component once and all three surfaces inherit it.
