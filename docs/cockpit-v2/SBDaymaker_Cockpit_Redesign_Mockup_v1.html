Design notes & recommendations — Queue

RECOne-press approve. Edit mode no longer has a separate save. Press E, fix the title/blurb in place (gold fields), toggle tags on/off, cycle the photo — then press A once. Approve commits every pending edit and publishes in the same action. A small gold banner on an edited card lists what changed so you can see the diff before committing.

RECHero flag from the queue (H). The ⭑ Hero pill marks hero_eligible and drops the item into the Hero plan shortlist at the moment you first see it — cheaper than hunting it down later in the catalog. It rides along with Approve; no extra step.

RECRestocked items carry a provenance pill. Anything that arrived via a Restock directive shows a small terracotta "restock · date_night" pill, so you know why it's in your queue and can judge it against the gap it was fetched to fill.

OPTReject reasons. R currently rejects instantly (cheap, reversible via Undo). If you later want rejection analytics ("what does the pipeline keep getting wrong?"), a two-key variant — R then 1/2/3 for off-brand / wrong facts / duplicate — is a small add. Deferred by default: it costs a keystroke on every reject.

Design notes & recommendations — Coverage

RECRelative shading + an absolute floor. You chose relative shading; the risk is a uniformly thin week where the "best" cell is 2 items and still shades green. The floor checkbox (on by default) forces any cell under a hard minimum to red regardless of rank. Pure relative is one click away if you prefer the cleaner read.

RECWindows are cumulative (45d contains 30d contains 14d…), so shading is computed within each column, never across the row — otherwise 45d would always out-green 7d and the grid would tell you nothing.

RECNeighborhood view uses the six nearby_zones, not the 11 granular neighborhoods — the zones are what actually power Near Me and the Discover SB guides, so they're the units worth stocking. Carpinteria/Riviera etc. roll up into the drilldown rows.

Restock & the batch-AI rule (§2.3). REC Default is a queued directive: pressing Restock writes a row ("find date_night things in the next 14 days") that tonight's existing pipeline picks up, runs through the exact same source catalogue → drop-gate → dedupe path, and lands results in tomorrow's queue. Zero new infrastructure, zero constraint bend. OPT "Run now" fires the same job on demand via a GitHub Action (~10–20 min); it's in the modal for genuinely urgent gaps, but every press is real API spend — the queued path should be your habit.

OPTA Tier-3 "depth" column (evergreen count per vibe/zone) could sit at the far left as a small grey number — it never changes week to week, which is why it's excluded from the RAG math, but it shows your fallback cushion. Say the word and I'll add it.

Design notes & recommendations — Live catalog

RECEdit → second look, without a gap on the site. You chose "edited card returns to the review queue." The important nuance: the previous approved version stays live while the edit waits — the site never shows a hole or an unreviewed draft. The row here gets a dashed gold border + "edit pending review" pill until you re-approve it in the Queue.

RECRows, not cards. The catalog can hold hundreds of things, so it's a dense scannable list (thumb chip · title · tier · when · vibes · zone), with the same ⭑ Hero toggle inline. Hero toggling is metadata-only, so it applies instantly — it doesn't round-trip through the queue.

OPTUnpublish. Each row's Edit sheet could carry an "Archive (remove from site)" action for closures and cancellations — one click, undoable. Recommended; cheap to add in the build.

Design notes & recommendations — Hero plan

RECPin is an override, not a requirement. Most days should stay on Auto — the ranker already picks well from the ⭑ pool, and hand-pinning 14 days is a time-budget trap. Pin the days that matter (a festival, a marquee show) and let Auto carry the rest. The dashed "Auto" tile shows tonight's projected pick so you can sanity-check without pinning.

RECThe picker only offers valid heroes: ⭑-flagged things that actually occur that day (or evergreens). It can't produce a hero pinned to a day its event doesn't exist — that class of error is designed out.

OPTIf a pinned event is cancelled or edited, the pin auto-clears back to Auto and the day flags amber in this rail — safer than a stale pin. This is the behavior the build should implement.

Restock

Everything found passes the same gate as the nightly run — deterministic start-time required, dedupe against the DB, category + zone checks — and arrives in your review queue. Nothing goes live without your approval.
