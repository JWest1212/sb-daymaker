The data catalogue — verified, costed at zero, ready to build

Every net-new source for SB Daymaker, prioritized into four waves. v3 replaces every assumed cost with a tested one: we fetched the doubtful sources without a browser and confirmed how each actually serves its data. The result — no source in this catalogue needs paid scraping.

What changed since v2

The whole cost column is now $0. Plain-fetch probes confirmed the sources we'd flagged as "Scrapfly" actually serve their data openly: Eventbrite, Lobero (and the venue pattern), MOXI (and the cultural pattern), Downtown SB, and AllEvents.in all render server-side; Visit SB is JavaScript-rendered but free through its WordPress API. Each row now shows its verified access route — API, iCal feed, or server-rendered scrape.

Scrapfly is retired to break-glass. It stays wired but unused; it's the fallback only if a source ever starts blocking. BroadwayWorld is dropped from the backstops — free but low-signal (its "Santa Barbara" region is mostly out-of-area). Visit SB and LiveNotes move from "managed scrape" to free (WordPress API; deterministic date parser).

02Cost — settled

Total new hard cost: $0 / month.

Every source resolves to an official API, a published iCal/RSS feed, or a server-rendered page a plain request can read — all on your existing GitHub Action infrastructure. The only credential to obtain is a free SeatGeek client_id (no card). Scrapfly remains wired as an unused break-glass fallback; it becomes a cost only if a source begins blocking, at which point the existing MANAGED_SCRAPE flag flips it on for that one source.

  • Edhat / Noozhawk — likely inside LiveNotes; prose times, low pass.
  • Per-brewery dated shows — Instagram-bound; covered by rhythms + submissions + Independent.
  • Facebook / Instagram (direct) — out by architecture.
  • Island Packers — departs Ventura, out of scope.
