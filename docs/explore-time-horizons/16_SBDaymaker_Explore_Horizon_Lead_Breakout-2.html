Explore — live frame,
three-horizon lead breakout

The live Explore tab is kept exactly as it ships. The one change: the dated lead section (Tier 1) — the only horizon-dependent block on the page — reflows into three distinct treatments by the original principle. Today keeps its editorial pick cards, This Week becomes a day-by-day rail, This Month becomes big-rock tiles. Everything else (hero, controls, Build-your-day CTA, the Recurring Weekly + Great any time tiers, footer) is shared and untouched.

Why the lead is the only place this applies: Tier 1 is the sole section driven by the horizon's date window (withinHorizon(thing, horizon, nowMs)). Tiers 2 and 3 are timeless — they render the same recurring and evergreen items no matter which horizon is selected — so giving them per-horizon layouts would be meaningless. The original "three UIs" principle maps cleanly onto exactly one block: the dated lead.

05 — The single switch

One horizon-aware branch in the lead

The whole change is contained in the Tier-1 lead renderer inside CascadeFeed. The header, the Build-your-day CTA, and Tiers 2 / 3 are untouched — they sit outside this function.

// CascadeFeed.tsx — only the LEAD (tier1) becomes horizon-aware.
// Header, Build CTA, Tier 2, Tier 3 stay exactly as they are today.
function LeadSection({ tier1, horizon }) {
  // sticky header unchanged: 📅 HORIZON_LABEL[horizon] + count badge
  if (horizon === 'today') return tier1.map(t => <PickCard t />);   // live
  if (horizon === 'week')  return <LeadDayRail items={tier1} />;        // new
  if (horizon === 'month') return <RockGrid items={tier1} />;       // new · soonest→furthest, 8 then "See more"
}
Risk surface is tiny. Two new presentational components (LeadDayRail, RockGrid — the latter wraps RockTile + the "See more" reveal) and a groupByDay helper. The only new state is a local monthExpanded boolean. No change to cascade(), withinHorizon(), filterByLens(), the hero, the controls, or the schema. The lead's data query is identical in all three branches — the 8-card cap is a render concern, not a filter.
