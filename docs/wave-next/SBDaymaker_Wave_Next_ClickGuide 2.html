Your build guide · the easy way

You copy one message. That’s the hard part.

No code. No commands. No typing things twelve times. You drop two files into a folder, paste one message to your assistant, and then just look at your screen and say “looks good” as it works through the changes.

The whole job, start to finish:

1. Put 2 files in a folder  →  2. Paste 1 message  →  3. Glance at your screen and say “looks good, next” (12 times)  →  4. Say “publish it.” Done.

You truly can’t break anything

The assistant works on a private copy of your site and saves a checkpoint after every step. If anything looks off, you say so in plain words and it fixes or undoes it. Nothing reaches your real website until you say you’re happy at the very end.

Step 1 Put the 2 files in your project

Here are the 2 files

1. SBDaymaker_Wave_Next_Build.md — the detailed plan. The assistant reads this (you don’t have to).

2. SBDaymaker_Wave_Next_ClickGuide.html — this guide, for you.

They both go in a new folder inside your project called docs/wave-next. The reason: the assistant can only read files that live inside your project folder. Here’s the easy way, right inside VS Code:

  1. Make sure your SB Daymaker project is open in VS Code. On the left you’ll see a list of your files and folders.Don’t see the list? Press Ctrl/Cmd + B to show it.
  2. Right‑click the top folder (your project’s name) → New Folder → type docs → press Enter.If a docs folder is already there, just use it.
  3. Right‑click your new docs folder → New Folder → type wave-next → press Enter.
  4. Drag both downloaded files from your Downloads onto the wave-next folder in that list, and let go.Prefer your normal file window? Right‑click wave-next → “Reveal in Finder” (Mac) or “Reveal in File Explorer” (Windows), then drag the files into the window that opens.
  5. You should now see docs › wave-next › SBDaymaker_Wave_Next_Build.md in the list. That’s it — Step 1 done. ✅

Step 2 Paste this one message to your assistant

Click into the Claude Code chat box (your AI assistant inside VS Code), paste the message below, and press Enter. This single message hands it everything — the setup, the preview, and all twelve changes.

📋 Copy all of this · paste it · press Enter

I’m a non‑technical founder, and you’ll be doing all the technical steps for me. I won’t use the terminal or type any commands. Please read the full plan at docs/wave-next/SBDaymaker_Wave_Next_Build.md.

Then handle everything yourself: make a safe private copy to work in (a branch called wave-next), install anything that’s needed, and start the live preview. Tell me, in plain words, the exact web address to open in my browser so I can watch.

After that, work through all twelve phases in order, one at a time. For each one: make the change, test it yourself, then tell me in simple terms what to look at on my screen — and wait for me to say it looks good before you save it and move to the next. Please do only one phase at a time, and never ask me to run commands or read code. When you reach the memory‑card step (Phase 9), pause and ask me the one question it needs.

Start now: confirm you’ve read the plan, give me the preview web address, and begin Phase 1.

What happens next

The assistant works for a minute, then gives you a web address (usually something like http://localhost:3000). Open it in your browser and keep that tab handy — it’s your live preview. Then it’ll start making the first change and tell you what to look at.

Step 3 Just look, and say one of these

From here on you don’t copy anything. After each change, the assistant tells you what to look at. You glance at your preview and reply with one of these:

✅
“Looks good — save it and let’s do the next one.”When it matches what you expected.
🔧
“Not quite — [say what’s off, like ‘the bottom bar is too dark’].”It’ll fix it, then you look again.
🐢
“Please slow down — one step at a time, and wait for my OK.”If it ever gets ahead of you.

Below is a simple cheat‑sheet of what each of the twelve changes does and what to look for — so you know what you’re approving. You don’t need to do anything with it except glance at it as you go.

⚠ One decision is coming at step 9 — here’s a heads‑up

When the assistant reaches the “Your Santa Barbara” memory card, it will pause and ask you one question. The card fills up as people mark places they’ve been, but we left out the gentle reminder that nudges them — so at first it’ll be mostly empty. You’ll choose:

  1. The simple version — show a friendly “Your map starts here” message until it fills up over time, or
  2. The welcome version — add a tiny one‑question welcome step so the card starts with one place already on it.

Either is fine. Just tell the assistant which you’d like when it asks. (A few other ideas we discussed are intentionally saved for later — they’re parked, not forgotten.)

Plain‑English words (only if you’re curious)

VS Code
The program on your computer where your website lives and gets built.
The assistant (Claude Code)
Your AI helper inside VS Code — the chat box you paste the one message into.
Preview
A private, live copy of your site running on your own computer (a web address like localhost:3000) so you can watch changes before anything is public.
Private copy / branch
A separate workspace where changes happen, so your real site is untouched until you’re ready. The assistant makes and manages it.
Save / checkpoint
A saved point after each step, so any one change can be undone without affecting the others.
Publish
The final step where your approved changes go live on the real website.
If you ever feel stuck

Type to the assistant: “I’m not sure what to do here — can you explain it simply and tell me the next step?” That always works. You’re in charge, and nothing is permanent until you publish.
