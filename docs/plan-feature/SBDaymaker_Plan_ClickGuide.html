Building the Plan feature — your kickoff guide

Everything you need to hand this to Claude Code and watch it build, one approvable phase at a time. No commands for you to run — just look and approve.

📄 1 message to paste 🧱 11 phases 👀 You approve each one

This pairs with two other files: SBDaymaker_Plan_Build.md (the detailed spec Claude Code follows) and SBDaymaker_Plan_Mockup.html (the picture it builds toward). You don't need to read those — Claude Code does. This guide is just for you.

⚠️ One thing to know before you start

Back in v9, you deliberately removed the old "My Plan" itinerary builder and timed plans — and your project's rulebook (CLAUDE.md) still says "do not build" them. The Plan feature is a fresh, intentional revival of that idea, so the very first thing Claude Code will do is update that rulebook to say Plan is now approved. That's Phase 0 below.

Why it matters: if we skipped this, Claude Code would (correctly) stop partway and ask why it's building something the rules forbid — wasting a session. Updating the rule first keeps the build smooth. You'll see the change and approve it like any other phase.

3Then just review and approve, phase by phase

Claude Code does the rest. Here's the rhythm you'll fall into.

  1. Claude Code builds one phase
    It works through a single chunk — for example, "the raised Plan button in the bottom bar."
  2. It stops and tells you what to look at
    It'll start the dev server and say something like "open the app and tap the new Plan button — here's what changed." It handles all the terminal, Git, and database work itself; it never asks you to type commands.
  3. You look — ideally on your phone
    Use your Vercel preview link or a narrow browser window. Check it feels right.
  4. You approve or redirect
    A short reply is all it needs (see below). Then it moves to the next phase.
To approve:  "Looks good — next."    To tweak:  "The Plan button is too big, shrink it, then continue."    To pause:  "Stop here for now."

4What you'll actually need to do

The 11 phases, in plain terms

So you know roughly where you are. Claude Code pauses after each for your OK.

0
Update the rulebook
Marks the Plan feature as approved in CLAUDE.md (the v9 reconciliation above). No app changes yet — just the diff to approve.
1
The raised Plan button
Adds the gold-and-terracotta "Plan" button to the bottom bar on every screen, and an empty Plan page behind it. Your three tabs stay exactly as they are.
2
The behind-the-scenes engine
No visible screen. Builds the day-planning logic, the saved-itinerary storage, and the curated "day shapes." Claude Code will show you it works via a quick log.
3
The setup screen
"Make My Day" plus your five questions and the "pin saved spots" hook. This is mockup Screen 1.
4
The itinerary spine
The signature screen — your day on a timeline that warms from morning teal to evening gold, with swappable stops. Mockup Screen 3.
5
Swap a stop
The sheet that lets you change any stop, with your saved spots floated to the top. Mockup Screen 4.
6
Build from your saved
The picker that pins your saved spots into the day and fills the gaps around them. Mockup Screen 2.
7
Save & the Days view
Saving a plan, the new "Things / Days" tabs on Saved, and the "My plans" dropdown in Plan. Mockup Screen 5.
8
Sharing a plan
The locked, view-only link a friend opens, with "Save this plan." Mockup Screen 6. (The one tiny database change lives here.)
9
Detail access & accessibility
Makes sure every card opens its detail the same way, plus the full keyboard/contrast/reduced-motion pass.
10
Edges, empties & final polish
Graceful "nothing saved yet" states, fallbacks so a day is never blank, loading states, and a final check against all six screens.

A note on the honest stuff. If at any phase something looks off to you, say so plainly — Claude Code will fix it before moving on, and that's far cheaper than catching it later. And if it ever flags a concern of its own (it's instructed to, rather than quietly working around a rule), take it seriously; that's the system working as intended.

When all 11 phases are done and approved, the Plan surface is live in your app — no accounts, no per-tap AI cost, fully on-brand. That's the whole build.
